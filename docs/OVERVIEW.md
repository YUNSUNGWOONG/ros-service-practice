## 1. 핵심 개념

클라이언트의 요청(Request)을 받아 서버가 로직을 처리한 뒤 응답(Response)을
반환하는 **ROS 2의 1:1 요청-응답 통신**을 다룹니다.

핵심 학습 요소:

- 사용자 정의 서비스 인터페이스(`.srv`) 설계와 빌드
- `create_service` / `create_client` 로 서버·클라이언트 구성
- `call_async` + `spin_once` + `future.done()` 기반 비차단(Non-blocking) 응답 처리
- `try/except` 로 서비스 통신 예외를 안전하게 처리

## 2. 실습 시나리오

**"항만 물류 로봇(AGV) 이동 구역 안전 승인 서비스"**

항만 내에서 화물을 운반하는 자율주행 물류 로봇이 특정 목적지로 이동하기 전,
관제 시스템(서버)에 해당 좌표의 이동 가능 여부를 질의하고 승인을 받는 시스템을
구현합니다. 별도의 물리 시뮬레이터(Isaac Sim 등)는 필요 없으며, 순수하게
노드 간 서비스 통신만으로 동작을 검증합니다.

## 3. 이 시나리오를 고른 이유

- 단순 Topic 통신(초급)을 넘어 **커스텀 srv 빌드 + 비동기 + 예외 처리**를
  한 번에 다루는 중급 주제라 서비스 통신의 전체 파이프라인을 체감할 수 있습니다.
- "좌표 승인"이라는 조건 검증 로직이 직관적이라, 통신 구조 자체에 집중할 수 있습니다.
